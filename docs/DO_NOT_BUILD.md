# DO NOT BUILD — speedandform (web + Field surfaces)

**Same bar as FORM iOS:** see **`FORM-iOS/docs/DO_NOT_BUILD.md`** for the full shared list (social mechanics, engagement loops, voice).

**Web-specific additions:**

- Turning The Field index into a **ranked** or “most active” feed
- Public comment threads that **change** how athletes write (performative audience) — if replies ship, they must match locked doctrine (group truth, no attention gaming); when unsure, **stop and ask**
- Breaking **production** `coach.html` or root static deploy without an explicit migration plan
- Duplicating athlete/session tables in Supabase without checking existing schema (audit first)

**Allowed:** structural tables, factual session rows, chronological Field stream per doctrine — when product + `form_web_spec_v2.md` say so.
